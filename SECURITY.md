\# Security Policy



\## Supported Versions



The latest version of GridSense Smart Electricity Monitoring Platform is actively maintained and receives security updates.



| Version        | Supported |

| -------------- | --------- |

| Latest         | ✅         |

| Older Versions | ❌         |



\## Reporting a Vulnerability



If you discover a security vulnerability in GridSense, please report it responsibly.



\### Contact



\* Email: \[saivamshidanthoju@gmail.com](mailto:saivamshidanthoju@gmail.com)



\### What to Include



Please provide:



\* Description of the vulnerability

\* Steps to reproduce the issue

\* Screenshots or proof of concept (if available)

\* Potential impact of the vulnerability



\### Response Process



After receiving a report:



1\. We will acknowledge the issue within 72 hours.

2\. We will investigate and validate the report.

3\. A fix will be developed and tested.

4\. The vulnerability will be resolved in a future update.



Please do not publicly disclose security vulnerabilities until they have been reviewed and addressed.



\## Security Features



GridSense includes:



\* Password hashing using bcrypt

\* JWT-based authentication

\* Environment variable protection using .env files

\* MongoDB secure database connectivity

\* Role-based access control for future releases

\* Secure API communication



Thank you for helping improve the security of GridSense.
